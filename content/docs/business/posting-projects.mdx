# Posting Projects

Creating clear, well-defined project postings is essential for attracting qualified students and ensuring successful collaborations.

## Before You Post

### Is Your Project Ready?

Ask yourself these questions:

âœ… **Is the scope well-defined?**
- Can you describe the project in 2-3 paragraphs?
- Are deliverables specific and measurable?
- Is the timeline realistic?

âœ… **Are requirements clear?**
- Do you know what skills are needed?
- Can you list must-have vs. nice-to-have qualifications?
- Is the time commitment realistic?

âœ… **Can you support the student?**
- Do you have time to provide guidance?
- Can you answer questions and give feedback?
- Are necessary resources available?

âœ… **Is this appropriate for students?**
- Can it be completed within the timeline?
- Is it a learning opportunity, not just grunt work?
- Does it provide real value to the student's portfolio?

> **ðŸ’¡ Tip**: If you answered "no" to any question, spend more time planning before posting!

### Project Size Guidelines

Choose the right scope for student projects:

| Project Type | Duration | Hours/Week | Best For |
|-------------|----------|------------|----------|
| **Micro** | 2-4 weeks | 5-10 hours | First-time projects, simple tasks |
| **Small** | 1-2 months | 10-15 hours | Standard projects, specific deliverables |
| **Medium** | 2-3 months | 15-20 hours | Complex projects, experienced students |
| **Large** | 3-6 months | 20+ hours | Major initiatives, advanced students |

> **âš ï¸ Warning**: Start with smaller projects until you're comfortable with the process!

#### 3. Project Description

- Expected outcomes

- Be concrete and measurable
- Include any milestones

## After Posting

### What Happens Next

1. **Review & Approval** (Immediate)
   - Post goes live immediately
   - Appears in student browse page
   - Email confirmation sent to you

   - Students can find your project via:
     - Browse page
     - Search
     - Category filter
     - Skill matching

3. **Applications** (1-14 days typically)
   - Students submit applications
   - You receive email notifications
   - Applications appear in your dashboard

## Best Practices

### Writing Effective Descriptions

**Do:**
- âœ… Use clear, concise language
- âœ… Break text into short paragraphs
- âœ… Use bullet points for lists
- âœ… Include specific, measurable deliverables
- âœ… Explain the "why" behind the project

**Don't:**
- âŒ Use excessive jargon or acronyms

- âŒ Be vague about expectations
- âŒ Demand professional-level work without support
- âŒ Make unrealistic timeline promises

### Attracting Quality Applications

**Clear Requirements = Better Matches**
- Specific skill requirements filter applicants
- Clear timeline prevents mismatched schedules
- Honest project scope attracts right skill level

**Sell the Opportunity**
- Emphasize learning and growth
- Highlight portfolio value
- Mention mentorship and support
- Explain real-world impact

**Be Authentic**
- Don't oversell or make false promises
- Be honest about challenges
- Admit if you're learning too
- Show your organization's personality

## Common Mistakes to Avoid

### âŒ Too Broad or Vague
**Problem:**
```
"Looking for a developer to help with our website and maybe some 
other projects. Must know coding."
```

**Better:**
```
"Redesign company homepage using React to improve mobile 
responsiveness and reduce bounce rate. Must have intermediate 
JavaScript and React skills."
```

### âŒ Unrealistic Scope
**Problem:**
```
"Build complete e-commerce platform with user accounts, payment 
processing, inventory management, and admin dashboard. Timeline: 
4 weeks, 10 hours/week."
```

**Why it's a problem:**
- Impossible timeline for scope
- Professional-level work for student hours
- No consideration for learning curve

**Better:**
```
"Develop prototype for e-commerce product catalog page using 
React and mock data. Timeline: 6 weeks, 12 hours/week. Full 
implementation will be handled by our team; this is for concept 
validation."
```

### âŒ Demanding Professional Work for Free
**Problem:**
```
"Need expert full-stack developer for mission-critical production 
application. Must have 3+ years experience. Unpaid."
```

**Why it's a problem:**
- Expects professional expertise
- Critical business application
- No compensation for demanding work

**Better:**
```
"Seeking intermediate student to build admin dashboard prototype 
for our internal tool. Great learning opportunity with senior 
developer mentorship. $750 stipend."
```

### âŒ Missing Key Information
**Problem:**
```
"Marketing help needed. Start ASAP."
```

**What's missing:**
- Specific tasks
- Required skills
- Timeline
- Deliverables
- Virtually everything!
